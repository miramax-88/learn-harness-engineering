# Documento de Qualidade (Quality Document)

Um instantâneo de qualidade para cada domínio de produto e camada arquitetônica. Tanto agentes quanto humanos podem usar este documento para entender rapidamente onde a base de código é forte e onde precisa de trabalho.

**Frequência de atualização:** Após cada sessão significativa, ou antes de iniciar uma nova fase de trabalho.

**Escala de avaliação:**

- **A**: Toda verificação passando, arquitetura limpa, legível para o agente, testes estáveis
- **B**: Verificação passando, majoritariamente limpa, pequenas lacunas na legibilidade ou cobertura de testes
- **C**: Funcionando parcialmente, lacunas conhecidas, algumas áreas de código difíceis de entender para os agentes
- **D**: Não está funcionando, ou possui problemas estruturais graves
